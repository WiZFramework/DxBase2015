ゲーム作成の準備


コンパイラと入力インターフェイス

　DxBase2015はVisuakStdio2013、2015でゲーム作成を行えます。作成したゲームは、Windows8.1、Windows10で動作します。
　奨励されるコンパイラはVisuakStdio2013です。
　もしお持ちでない方はMicrsoft社のサイトから、Visual Studio Community 2013 もしくはVisual Studio Community 2015 をダウンロードしてください。
　また、セットアップの際VisualC++とWindowsSDKを選択してください。

　チュートリアルやサンプルでは、ほとんどがXBox360コントローラでの操作になっています。キーボード及びマウス入力を得るには、キーボード及びマウス入力のサンプルをご確認ください。

ダウンロードと解凍

　DxBase2015はGitHubに公開されています。
　アドレスは https://github.com/WiZFramework/DxBase2015
　です。
　上記サイトにアクセスすると、DownloadZIPというボタンがありますので、そこで最新版をダウンロードしてください。解凍すると以下のようなディレクトリ構成になります。

 

図setup_a

 

　Documentディレクトリには、このページを含めたドキュメントが入ってます。index.htmlをブラウザで開いてください。
　Libsディレクトリには、フレームワーク本体や、DirectXTKライブラリなどが入ってます。
　Libs/includeとLibs/libディレクトリには、このあと、Autodesk社のFBX SDK（FBX Software Development Kit）のヘッダとライブラリとコピーしていただきます。（FBXSDKのセットアップで解説します）
　Tutorial001など、Tutorialではじまるディレクトリには、チュートリアルサンプルが入ってます。チュートリアルの解説には、このドキュメントにあります。
　Sample001などのディレクトリにはサンプルが入ってます、内容は、ドキュメントのindex.htmlを見てください。チュートリアル以外のサンプルにも、できるだけドキュメントを記述してますので、参考にしてください。

FBXSDKのセットアップ

　GitHubからのダウンロードには、FBXモデルの読み込みのためのライブラリが含まれてません。以下の手順でセットアップしたのち、必要なファイルを、DxBase2015内に配置して利用します。
  • Autodesk社のFBX SDK（FBX Software Development Kit）をセットアップします。
  • FBX SDKは検索エンジンなどで　FBX SDK　などで検索するとたどり着けます。そこからダウンロードのページに移り、ご使用のコンパイラに合わせたバージョンのFBX_SDKをダウンロードしてください。（ご使用のコンパイラのバージョンと、FBX_SDKのバージョンが合わないと正常にビルドできませんので注意してください）。
  • 現在、動作が確認されているFBX_SDKのバージョンは、VisualStdioS2013であれば、FBX SDK 2015.1 VS2013、FBX SDK 2016.0 VS2013 、FBX SDK 2016.1 VS2013 です。
  • VisualStdioS2015であれば、FBX SDK 2016.1 VS2015 です。
  • 2015年8月時点でのFBX SDKの最新バージョンはFBX SDK 2016.1です。こちらは、VisualStdioS2013、VisualStdioS2015両方で動作確認できています。
  • FBX SDKは実行ファイルになっていますので、そのままセットアップ可能です。
  • FBX SDKのセットアップが済んだら、セットアップされたディレクトリ
    （デフォルトで「C:\Program Files\Autodesk\FBX\FBX SDK\2016.1」などにエクスプローラで移動してください。
  • FBX SDKがセットアップされたディレクトリの「include」ディレクトリ内の、fbxsdk.hというファイルと、fbxsdkディレクトリを、そのまま、DxBase2015を解答したディレクトリの「Libs\include」内にコピーしてください。
  • FBX SDKがセットアップされたディレクトリの「lib\vs2013\x86\debug」あるいは「lib\vs2015\x86\debug」ディレクトリ内の、libfbxsdk-mt.libというファイルと、libfbxsdk.pdbを、DxBase2015を解答したディレクトリの「Libs\lib\Debug」内にコピーしてください。
  • FBX SDKがセットアップされたディレクトリの「lib\vs2013\x86\release」あるいは「lib\vs2015\x86\release」ディレクトリ内の、libfbxsdk-mt.libというファイルを、DxBase2015を解答したディレクトリの「Libs\lib\Release」内にコピーしてください。
  • x64（64ビット版）には対応しておりません。必ずx86（32ビット版）をセットアップしてください。

　以上で、セットアップは完了です。

サンプルの起動

　各チュートリアルやサンプルディレクトリにはソリューションファイル(DxBase2015Desktop.sln)が入っています。サンプルごとにソリューションがありますので、別々に起動可能です。
　別々に起動は可能ですが、ライブラリ等、共有部分も多いので、1つのサンプルを実行したら、別のサンプルを起動する前に先の起動は閉じておくことをおすすめします。
　ソリューションファイル(DxBase2015Desktop.sln)は、VisualStdio2013用のソリューションになっています。
　ソリューションを開くことで、各VisualStdioを起動できるわけですが、GitHubからダウンロードした直後はセキュリティの確認が出ます。ここは、是非こちらを信頼していただいて、はいで作業を進めてください。
　VisualStdio2013のみセットアップされてる環境では、ダブルクリックでVisualStdio2013が起動します。
　VisualStdio2015のみセットアップされてる環境では、ダブルクリックでVisualStdio2015が起動し、バージョンアップするかどうかの確認が出ますので、そのままバージョンアップしてください。FBX_SDKがVisualStdio2015用のがセットされていれば問題なくリビルドできます。
　VisualStdio2013とVisualStdio2015両方セットアップされている環境では、ダブルクリックで開くとVisualStdio2013が起動します。もし、VisualStdio2015で作業を進めたければ、ソリューションを右ボタンでクリックし、プログラムから開くでVisualStdio2015を選択してください。バージョンアップするかどうかの確認が出ますのではいを選択してください。FBX_SDKがVisualStdio2015用のがセットされていれば問題なくリビルドできます。
